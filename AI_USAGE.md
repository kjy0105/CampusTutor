# AI Usage Report

## AI Tools Used
- Antigravity (Google DeepMind Coding Assistant)
- VS Code (Workspace Integration)

## Tasks Supported by AI
- **중복 수강 방지 로직 설계 및 구현**: 한 계정에서 동일한 선생님에게 중복 수강이 되지 않도록 차단하고, 동일 과목 신청 시 경고 팝업창을 띄우는 기능 추가.
- **복수 과목 병렬 UI 구현**: 동일 선생님에게 다른 과목을 신청할 때는 별도의 채팅방이 여러 개 생성되는 것이 아니라, 하나의 채팅창 내에 수강 중인 과목들이 병렬로 표시되도록 UI/UX 고도화.
- **학생 대시보드 필터링**: 학생 대시보드 화면에서 현재 수강 중인 과목들만 필터링하여 일관되게 보여주도록 개선.
- **과외 역제안(Reverse Proposal) 시스템 설계**: 학생이 공개 요청 풀에 과외 요청을 올리면 선생님들이 조건(시급, 소개글)을 기재하여 역제안을 보낼 수 있는 흐름 설계 및 역제안 시 다수 과목 중 1개를 골라 제안하는 로직 구현.
- **알림센터 딥링크(Deep-linking) 구현**: 알림을 눌렀을 때 알림 성격에 따라 채팅방 자동 팝업, 대시보드 페이지 전환, 마이페이지 수업 관리 탭 자동 전환 등이 동작하는 내비게이션 기능 추가.
- **선생님 대시보드 및 실시간 연동 디버깅**: 역제안 수락 후 선생님 대시보드 및 홈 화면에서 매칭 카드가 사라지던 현상 분석 및 실시간 채팅방 ID(`roomId`) 공백 유실 오류 디버깅.
- **신규 선생님 가입 시 지역 정보 누락 버그 수정**: 선생님 회원가입 과정에서 거주지역(`region`, `subRegion`) 데이터가 `users` 노드에 저장되지 않아 매칭 연동이 끊어지는 결함 해결.

## Example Prompts
1. *"한 계정에서 같은 선생님에게 중복 수강이 되지 않도록 해줘. 동일 과목 신청 시 차단 팝업창을 띄워주고, 다른 과목을 신청할 때는 한 채팅창 위에 수강하는 과목이 병렬로 뜨게 해줘."*
2. *"선생님이 학생의 공개 요청 건에 대해 역제안할 수 있는 시스템을 만들어줘. 학생이 여러 과목을 요청했다면 선생님이 역제안할 때 그 중 한 과목을 선택할 수 있도록 설계해줘."*
3. *"선생님이 역제안을 해서 매칭이 완료되었는데, 선생님 대시보드에 채팅방이 생성되지 않고 매칭된 진행 수업 카드조차 뜨지 않는 이유를 찾아서 해결해줘."*
4. *"알림창에서 알림을 클릭하면 해당하는 페이지나 탭으로 즉시 이동하게 딥링크를 추가해줘. 채팅 알림을 누르면 즉시 채팅 모달이 켜지게 연동해줘."*

## AI Outputs We Modified
- **데이터 정제 로직 보강**: 중복 수강 검사식 및 매칭 필터 식에서 `.trim()` 처리가 누락되어 발생하던 공백 불일치 예외 처리 추가.
- **대화방 개설 Key(`roomId`) 함수 개선**: 방 ID 생성 시 참여자들의 이름에서 앞뒤 공백을 철저히 제거하여 양방향 일관성 보장.
- **Firebase 회원가입 스키마 수정**: `handleSignup` 내 `userData`에 `region`과 `subRegion`을 바인딩하여 데이터가 누락 없이 `users` 노드에 탑재되도록 변경.
- **동적 그룹화 로직 수정**: `updateDashboard()` 및 `updateTutoringList()` 내의 `groupedAccepted` 연산에서 학생 이름을 기준으로 병렬 과목을 묶을 때 이름의 공백 오류를 막기 위해 trim된 문자열을 키로 사용하도록 수정.

## Core Files We Can Explain
- [index.html]: 전체 매칭 플랫폼의 UI 뼈대, 회원가입 폼(선생님/학생 세분화), 실시간 알림 드로어 및 1:1 실시간 채팅 레이아웃 구조 정의.
- [script.js]: Firebase Realtime Database의 실시간 이벤트 리스너 연동, 비즈니스 로직(중복 체크, 역제안 수락 및 스키마 클리업, 포인트 보너스 적립, 알림 내비게이션) 제어.

## What We Learned
- **데이터 무결성 및 정제(Sanitization)의 필수성**: 사용자 이름 데이터 뒤의 작은 공백(`"박건우 "` vs `"박건우"`) 하나가 데이터베이스 키 매칭 실패 및 대화방 해시 불일치를 야기하여 서비스 단절을 가져올 수 있음을 깨닫고, 폼 입력 시점과 비교 시점에 철저하게 공백 제거(`.trim()`)를 적용해야 한다는 소중한 교훈을 얻었습니다.
- **상태 동기화와 스키마 설계**: 실시간 데이터베이스 환경에서 동시다발적인 쓰기가 일어날 때(역제안 수락 및 타 제안 일괄 삭제 등) 데이터의 유기적 관계와 흐름을 시각적으로 정형화하는 능력을 키웠습니다.
- **AI 도구의 주체적 제어 및 검증**: AI가 생성한 비즈니스 로직이나 DB 쿼리는 초기 가이드를 잡는 데 훌륭하지만, 최종적인 결합 테스트와 세부 예외 상황(가입 시 누락 필드 확인 등) 분석 및 실배포 환경 검증은 개발자가 주체적으로 확인하고 수정해야 높은 품질의 소프트웨어를 완성할 수 있음을 체감했습니다.
